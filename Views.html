Lec - 17

Views

  • Virtual tables for sqls(if not specified, won't be stored physically )
  • Are created from our base tables
  • Whatever we add in base table, is available in view and vice-versa
    • Cannot add record from view to base if the view does not contain some field which has not null constraint

Use cases

  1. Providing restricted access to some users
  2. For better performance of join queries which are executed frequently
    • works like cache, once we join some tables we can store it usign a view and the access again

Syntax and examples
